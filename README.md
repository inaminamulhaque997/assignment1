1. What are some differences between interfaces and types in TypeScript?

TypeScript এ interface এবং type alias এর মধ্যে বেশ কিছু মিল থাকলেও তাদের ব্যবহারে কিছু মৌলিক পার্থক্য রয়েছে:
Declaration Merging: একই নামে যদি একাধিক interface তৈরি করা হয়, TypeScript সেগুলোকে স্বয়ংক্রিয়ভাবে একত্রিত (merge) করে একটি ইন্টারফেসে পরিণত করে। কিন্তু type এর ক্ষেত্রে এটি সম্ভব নয়; একই নামে একাধিক type declare করলে এরর দেখাবে।
Use Cases: সাধারণত Object এর গঠন বা Class এর স্ট্রাকচার ডিফাইন করার জন্য interface বেশি ব্যবহার করা হয়। অন্যদিকে, Union types, Tuple বা primitive types এর জন্য type alias ব্যবহার করা উত্তম।



3. Explain the difference between any, unknown, and never types in TypeScript.
TypeScript এ এই তিনটি টাইপ বিশেষ বিশেষ ক্ষেত্রে ব্যবহৃত হয়:
any: এটি TypeScript এর টাইপ চেকিং সিস্টেমকে বন্ধ করে দেয়। কোনো ভেরিয়েবলের টাইপ any হলে, আপনি তাতে যেকোনো ভ্যালু অ্যাসাইন করতে পারেন এবং যেকোনো মেথড কল করতে পারেন (তা বাস্তবে থাকুক বা না থাকুক)। এটি ব্যবহার করা সাধারণত নিরুৎসাহিত করা হয় কারণ এটি TypeScript ব্যবহারের মূল উদ্দেশ্যকেই ব্যাহত করে।
unknown: এটি any এর একটি নিরাপদ সংস্করণ। আপনি unknown টাইপের ভেরিয়েবল এ যেকোনো ভ্যালু রাখতে পারেন, কিন্তু টাইপ চেক (Type Guard) না করা পর্যন্ত সেই ভেরিয়েবল দিয়ে কোনো অপারেশন (যেমন মেথড কল বা প্রপার্টি অ্যাক্সেস) করা যায় না। এটি কোডকে any এর চেয়ে বেশি নিরাপদ রাখে।
never: এটি এমন একটি টাইপ যা নির্দেশ করে যে কোনো ভ্যালু কখনোই আসবে না। সাধারণত কোনো ফাংশন যদি সব সময় Error throw করে অথবা ইনফাইনাইট লুপে চলে যায় (অর্থাৎ ফাংশনটি কখনো শেষ হয় না), তখন তার রিটার্ন টাইপ হয় never।
